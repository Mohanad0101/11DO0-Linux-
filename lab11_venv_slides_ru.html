Проблема
Два проекта.
Один Python. Один конфликт.
Python хранит только одну версию каждого пакета. Установка новой версии автоматически удаляет старую — и ломает проект, который на неё рассчитывал.

📁 Новый проект
Flask == 3.0

Начат сегодня. Требует актуальных функций Flask 3 и исправлений безопасности.
Установить Flask 3.0: pip автоматически удалит Flask 2.3 — старый проект перестанет работать.
Оставить Flask 2.3: новый проект не получит нужные функции. Код придётся переписывать.
Третьего варианта нет — без изоляции приходится выбирать, кто пострадает.
Решение
Каждый проект получает
свой личный Python
Виртуальная среда — это папка внутри проекта, содержащая собственный интерпретатор Python и собственный pip. Пакеты, установленные в неё, невидимы для всех остальных проектов.

📁 new_project/

🐍 python (свой)

Имеет Flask 3.0 — и никак не влияет на старый проект.
Создать среду
$ python3 -m venv .venv
Создаёт папку .venv/ внутри проекта со своим Python и pip.
Активировать
$ source .venv/bin/activate (.venv) user@host:~$
Префикс (.venv) в строке приглашения подтверждает, что среда активна.
Выйти из среды
(.venv) $ deactivate $ # строка приглашения обычная
Всегда деактивируйте среду перед переходом к другому проекту.
Вторая проблема — Переносимость
На вашем компьютере работает.
На другом — нет.
Даже если конфликтов нет, без записи зависимостей ни коллега, ни вы сами на другом компьютере не сможете воспроизвести рабочую среду.

✗ Компьютер коллеги — ошибки
$ python app.py ModuleNotFoundError: Нет модуля с именем 'flask' $ pip install flask && python app.py ImportError: cannot import 'Markup' # неверная версия Werkzeug ... часы отладки ...
Причина: файл requirements.txt не был сохранён. Коллега не знает, какие версии пакетов вы использовали.
Решение — две команды
# Вы: сохранить среду в файл $ pip freeze > requirements.txt # Коллега: точно воспроизвести её $ pip install -r requirements.txt
Как выглядит requirements.txt
Flask==3.0.3
Werkzeug==3.0.3
Jinja2==3.1.4
... точные версии, зафиксированы
