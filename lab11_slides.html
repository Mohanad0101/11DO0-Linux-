Virtual Environments — Workflow
The venv Lifecycle
Five steps — performed once per project, in this order, every time you start a new Python project.
1
CREATE
python3 -m venv .venv
Creates a hidden .venv/ folder containing a private interpreter, pip, and an empty site-packages.
→
2
ACTIVATE
source .venv/bin/activate
Prepends .venv/bin to PATH. Prompt shows (.venv). Now python and pip point to the venv.
→
3
INSTALL
pip install numpy
Packages install into .venv/lib/, not the system. The system Python is completely untouched.
→
4
USE
python hello.py
Run scripts normally. They import from .venv's site-packages, not the system's.
→
5
DEACTIVATE
deactivate
Restores the original PATH. The (.venv) prefix disappears. Always deactivate before switching projects.
What lives inside .venv/
.venv/ ├── bin/ │ ├── python ← private interpreter │ ├── pip ← private pip │ └── activate ← the script you source ├── lib/ │ └── python3.x/ │ └── site-packages/ ← installed packages go here └── pyvenv.cfg ← records Python version
Key rule
Never switch projects without running deactivate first.

Using the wrong venv means the wrong Python version and the wrong package versions — errors that are hard to debug.

shutil + pathlib in action
import shutil from pathlib import Path src = Path("workspace/note.txt") dst = Path("workspace/docs/note_copy.txt") # Ensure destination folder exists dst.parent.mkdir(parents=True, exist_ok=True) # Copy (preserves timestamps) shutil.copy2(src, dst) # Move / rename shutil.move(str(src), "workspace/renamed.txt") # Delete Path("workspace/renamed.txt").unlink()
Why shutil.copy2 not shutil.copy?
copy2 preserves the source file's metadata (modification time, permissions). Always prefer copy2 in file manager applications.
Why shutil.move not Path.rename?
Path.rename fails when moving across different filesystem partitions. shutil.move handles both cases correctly.
Security — Path Safety (Case Study 1)
The Sandbox: Blocking Path Escape
Every user-supplied path is validated before any I/O. Paths that leave the workspace are rejected with PermissionError.
Filesystem View
/home/user/lab11/fm_project/
├── config.json
├── safe_paths.py
└── workspace/ ← sandbox boundary
├── note.txt ✓ OK
└── folder/a.txt ✓ OK
/home/user/lab11/
└── secret.txt ✗ BLOCKED
/etc/
└── passwd ✗ BLOCKED
✓ "note.txt" → resolves inside workspace
✓ "folder/a.txt" → subdirectory, still inside
✗ "../secret.txt" → escapes via ..
✗ "/etc/passwd" → absolute path, outside
Sandbox.resolve_inside() — the guard
def resolve_inside(self, user_path: str) -> Path: # Step 1: join with workspace, then resolve candidate = (self.workspace / user_path).resolve() # Step 2: containment check if (candidate == self.workspace or self.workspace in candidate.parents): return candidate # ✓ allowed # Step 3: anything else is outside — reject raise PermissionError( f"Access denied: '{user_path}' " "resolves outside the workspace." )
Containment rule:
A path is inside the workspace if the workspace is either equal to the candidate, or appears in the candidate's chain of parent directories.

self.workspace in candidate.parents
Security — Path Safety Detail
How Path.resolve() Eliminates ..
Path.resolve() converts any path to an absolute, canonical form — making directory traversal attacks impossible to hide.
✗ Escape attempt
workspace/../secret.txt
↓ Path.resolve() — step by step
Start: workspace/../secret.txt
→ Join with cwd:
  /home/user/lab11/fm_project/workspace/../secret.txt
→ Expand ".." (go one level up):
  /home/user/lab11/fm_project/secret.txt
/home/user/lab11/fm_project/secret.txt
workspace/ is NOT in this path's parents → BLOCKED ✗
✓ Safe path
workspace/docs/note.txt
↓ Path.resolve() — step by step
Start: workspace/docs/note.txt
→ Join with cwd:
  /home/user/lab11/fm_project/workspace/docs/note.txt
→ No .. to expand.
  Path is already clean.
/home/user/lab11/fm_project/workspace/docs/note.txt
workspace/ IS in candidate.parents → ALLOWED ✓
What Path.resolve() does
1
Expands .. and .
Removes every "go up one level" component.
2
Follows symlinks
A symlink pointing outside the workspace is also caught.
3
Returns absolute path
Result always starts from / — unambiguous for comparison.
After resolve(), path comparison becomes simple string/object equality — no need to parse path components manually.
