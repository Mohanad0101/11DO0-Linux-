## 1. Введение

Linux широко используется в системном администрировании и разработке благодаря надёжной командной строке и единой модели файловой системы. **Python** хорошо подходит для системного программирования в Linux: в стандартной библиотеке есть средства для работы с файлами и каталогами, текстом, конфигурацией и автоматизацией.

В этой работе вы подготовитесь к созданию **простого консольного файлового менеджера**. Вы освоите:

- **Безопасную работу с путями** ограничение всех операций заданной рабочей областью
- **Базовые файловые операции**  чтение, запись, копирование, перемещение, удаление
- **Воспроизводимые окружения**  виртуальные среды и `requirements.txt`
- **Структурированную конфигурацию**  вынесение настроек в `config.json`

В результате вы получите минимальный, но полноценный файловый менеджер с изолированной рабочей областью, который можно запускать и проверять из командной строки.

### Шаг A3: Базовые операции с файлами

> **Текущее расположение:** `~/lab11`

**Что сделать:**

```bash
touch sample.txt
ls -la
cp sample.txt sample_copy.txt
mv sample_copy.txt renamed.txt
rm renamed.txt sample.txt
ls -la
```

**Зачем:** Эти четыре команды соответствуют основным операциям, которые должен выполнять ваш файловый менеджер на Python:

| Команда оболочки | Операция | Эквивалент в Python |
|------------------|----------|----------------------|
| `touch file` | Создать пустой файл | `Path.write_text("")` |
| `cp a b` | Копировать | `shutil.copy2(a, b)` |
| `mv a b` | Переместить / переименовать | `shutil.move(a, b)` |
| `rm file` | Удалить | `Path.unlink()` |

Последняя команда `ls -la` подтверждает, что каталог снова пуст.


**Зачем:** Если файла `notes.txt` нет, Vim создаст его при сохранении. Vim запускается в **нормальном режиме**; ввод текста возможен только после перехода в **режим вставки**.

---

### Шаг B2: Минимальный цикл: правка — сохранение — выход

При каждом редактировании файла в Vim выполняйте четыре шага:

| Шаг | Клавиша / команда | Действие |
|-----|-------------------|----------|
| 1 | Нажать `i` | Войти в **режим вставки**. В строке состояния отображается `-- INSERT --`. |
| 2 | Ввести (или вставить) текст | Текст добавляется в позиции курсора. |
| 3 | Нажать `Esc` | Вернуться в **нормальный режим**. Строка состояния очищается. |
| 4 | Ввести `:wq` и `Enter` | **Записать** файл на диск и **выйти** из Vim. |

**Зачем:** В нормальном режиме выполняются навигация и команды; в режиме вставки — ввод текста. Перед вводом `:wq` нужно обязательно быть в нормальном режиме.

**Практика:** Введите строку текста, сохраните и выйдите. Проверьте создание файла:

```bash
cat notes.txt
```

Удалите пробный файл по окончании: `rm notes.txt`

---

### Краткая справка Vim — сохранение и режимы

| Действие | Команда | Примечание |
|----------|---------|------------|
| Режим вставки | `i` | Ввод с позиции курсора |
| Вставка после курсора | `a` | Ввод после текущего символа |
| Новая строка ниже | `o` | Новая строка и переход в режим вставки |
| Нормальный режим | `Esc` | Перед любой командой нажимайте Esc |
| Сохранить | `:w` + `Enter` | Продолжить редактирование |
| Сохранить и выйти | `:wq` + `Enter` | Обычное завершение работы |
| Выйти без сохранения | `:q!` + `Enter` | Все изменения отменяются |
| Режим вставки из буфера | `:set paste` + `Enter` | Удобно при вставке кода из буфера |
| Выключить режим вставки из буфера | `:set nopaste` + `Enter` | Вернуть обычное поведение отступов |

> **Совет при вставке:** При вставке многострочного кода из терминала или браузера сначала введите `:set paste`, нажмите `i`, вставьте текст, затем `Esc`, `:set nopaste` и при необходимости `:wq`. Так Vim не испортит отступы.

### Краткая справка Vim — навигация

| Действие | Команда | Примечание |
|----------|---------|------------|
| Влево / вниз / вверх / вправо | `h` / `j` / `k` / `l` | Работают и стрелки |
| Следующее / предыдущее слово | `w` / `b` | Переход по словам |
| Начало строки | `0` | Первый символ строки |
| Первый непробельный символ | `^` | Удобно в коде с отступами |
| Конец строки | `$` | Последний символ строки |
| Начало файла | `gg` | Первая строка |
| Конец файла | `G` | Последняя строка |
| Перейти на строку N | `Ngg` или `:N` | Например `25gg` или `:25` |

### Краткая справка Vim — правка и поиск

| Действие | Команда | Примечание |
|----------|---------|------------|
| Отменить | `u` | Отмена последнего изменения |
| Повторить | `Ctrl+r` | Вернуть отменённое |
| Удалить текущую строку | `dd` | Строка попадает в буфер |
| Удалить до конца строки | `D` | От курсора до конца строки |
| Копировать строку | `yy` | Вставка — командой `p` |
| Вставить после курсора | `p` | Вставка из буфера |
| Поиск вперёд | `/слово` + `Enter` | Подсветка совпадений |
| Следующее совпадение | `n` | Повтор — следующее |
| Заменить все вхождения | `:%s/старое/новое/g` + `Enter` | Замена по всему файлу |
| Заменить с подтверждением | `:%s/старое/новое/gc` + `Enter` | Запрос перед каждой заменой |

## Часть D — Виртуальные среды и воспроизводимая установка

> **Текущее расположение:** `~/lab11`

**Виртуальная среда** (venv) — это изолированное окружение Python, в котором пакеты проекта отделены от системного Python и от других проектов. Это предотвращает конфликты версий.

Файл **`requirements.txt`** фиксирует точные имена и версии пакетов в venv. То же окружение можно воссоздать на другой машине командой:

```bash
pip install -r requirements.txt
```

Эта команда читает `requirements.txt` построчно и устанавливает перечисленные пакеты в активную виртуальную среду.

## Часть E — Правильное копирование проекта (только код и requirements)

**Принцип:** Виртуальная среда (папка `.venv/`) **не переносима**: в ней записаны абсолютные пути конкретной машины и каталога. При копировании проекта — в новую папку, на флешку или другой компьютер — копируйте только:

- Исходные файлы `.py`

В таблице ниже сопоставлены типичные операции в оболочке с инструментами стандартной библиотеки Python, которые понадобятся в файловом менеджере. Изучите таблицу перед написанием кода в части I.

## Часть H — Файл конфигурации (config.json)

> **Текущее расположение:** `~/lab11`

### H1) Зачем нужен файл конфигурации?

В коде файлового менеджера не должны быть жёстко прописаны пути, кодировка или флаги. При изменении таких значений пришлось бы править код. **Файл конфигурации** выносит их наружу: меняется файл, а не код.

В этой работе в `config.json` задаётся **рабочая область** (workspace) — граница каталога, внутри которой разрешены все файловые операции. Чтобы направить менеджер в другую папку, достаточно изменить `"workspace"` в JSON.

---

### H2) Создать каталог fm_project и config.json

**Что сделать:**

```bash

mkdir -p fm_project
cd fm_project
```

Создайте файл `config.json`:

1. Откройте Vim: `vim config.json`
2. Введите `:set paste` + `Enter`, затем `i`, вставьте содержимое ниже, затем `Esc`, `:set nopaste`, `:wq`, `Enter`.

```json
{
  "workspace": "workspace",
  "start_dir": ".",
  "encoding": "utf-8",
  "allow_delete": true
}
```

**Назначение полей:**

| Поле | Тип | Значение |
|------|-----|----------|
| `workspace` | строка | Путь к папке песочницы **относительно fm_project**. Все файловые операции ограничены этой папкой. |
| `start_dir` | строка | Начальная подпапка внутри рабочей области при старте. `"."` — корень рабочей области. |
| `encoding` | строка | Кодировка при чтении и записи текстовых файлов (например `"utf-8"`). На чтение самого `config.json` не влияет — JSON всегда в UTF-8. |
| `allow_delete` | логическое | Флаг. При `false` команда удаления отключена; администратор может запретить удаление без правки кода. |

**Ссылка:** [json — кодирование и декодирование JSON](https://docs.python.org/3/library/json.html)

#### З1 — Шаг 3: Создать config_loader.py

> **Расположение:** `~/lab11/fm_project`

**Что сделать:** Откройте в Vim файл `config_loader.py` и вставьте код из английской версии лабораторной работы (или следующий код):

```python
import json
from dataclasses import dataclass
from pathlib import Path


@dataclass(frozen=True)
class AppConfig:
    workspace: Path       # абсолютный или относительный путь к папке песочницы
    start_dir: str = "."  # начальная подпапка внутри рабочей области
    encoding: str = "utf-8"
    allow_delete: bool = True


def load_config(path: str = "config.json") -> AppConfig:
    """
    Читает config.json из текущего рабочего каталога и возвращает AppConfig.
    Вызывает FileNotFoundError, если файл отсутствует.
    Вызывает ValueError, если отсутствует обязательное поле 'workspace'.
    """
    p = Path(path)
    if not p.exists():
        raise FileNotFoundError(
            f"Файл конфигурации не найден: '{path}'. "
            f"Запускайте скрипт из каталога fm_project."
        )

    data = json.loads(p.read_text(encoding="utf-8"))

    if "workspace" not in data:
        raise ValueError("В config.json должно быть поле 'workspace'.")

    return AppConfig(
        workspace=Path(data["workspace"]),
        start_dir=data.get("start_dir", "."),
        encoding=data.get("encoding", "utf-8"),
        allow_delete=bool(data.get("allow_delete", True)),
    )
```

**Зачем:** Вынос загрузки конфигурации упрощает поддержку: при смене пути или кодировки правится только `config.json`. Декоратор `@dataclass(frozen=True)` делает `AppConfig` неизменяемым после загрузки.

## Типичные проблемы и решения

| Симптом | Вероятная причина | Решение |
|---------|-------------------|---------|
| `FileNotFoundError: Config file not found: 'config.json'` | Скрипт запущен не из `fm_project`. | Выполните `cd ~/lab11/fm_project` и запустите снова. |
| `PermissionError: Access denied: '...' resolves outside the workspace.` | Введённый путь выходит за пределы рабочей области. | Используйте только относительные пути внутри workspace (например `notes.txt`, `subdir/file.txt`). Это ожидаемое поведение безопасности. |
| `ModuleNotFoundError: No module named 'config_loader'` | Скрипт запущен из другого каталога. | Убедитесь, что `pwd` — `~/lab11/fm_project`. Все модули `.py` должны быть в одном каталоге. |
| `python3: command not found` или `pip3: command not found` | Инструменты не установлены или не в PATH. | Выполните часть C заново. Проверьте: `which python3`. |
| После `source .venv/bin/activate` в строке приглашения нет `(.venv)` | Используется другая оболочка (например fish или csh). | Запустите `bash` и снова выполните команду активации. |
| При вставке в Vim появляются лишние символы или сбиваются отступы | Режим вставки конфликтует с буфером. | Перед нажатием `i` введите `:set paste`, вставьте текст, затем `:set nopaste` перед сохранением. |
| При выполнении команды выводится `Usage: copy <src> <dst>` | Указано слишком мало аргументов. | Укажите и источник, и приёмник: `copy a.txt b.txt`. |
| `flask: command not found` | venv Flask не активирована или Flask не установлен. | Выполните `source ~/lab11/flask_blog_practice/.venv/bin/activate` или используйте `python -m flask run`. |
| `OSError: [Errno 98] Address already in use` на порту 5000 | Уже запущен другой процесс Flask. | Остановите его через `Ctrl+C` в другом терминале или запустите на другом порту: `flask run --port 5001`. |
| `pip install numpy` падает с сетевыми ошибками | Нет доступа в интернет. | Запросите у преподавателя офлайн-файл `.whl` и установите: `pip install numpy-<версия>.whl`. |
| `git clone` завершается с «Repository not found» | Неверный URL или нет доступа. | Проверьте URL. Для частных репозиториев нужна авторизация или SSH-ключ. |
