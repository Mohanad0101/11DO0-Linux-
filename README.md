### Шаг A3: Практика базовых файловых операций

> **Текущее расположение:** `~/lab11`

**Что нужно сделать:**

```bash
touch sample.txt
ls -la
cp sample.txt sample_copy.txt
mv sample_copy.txt renamed.txt
rm renamed.txt sample.txt
ls -la
```

**Зачем:** Эти четыре команды напрямую соответствуют базовым операциям, которые ваш файловый менеджер на Python должен реализовать:

| Команда оболочки | Операция | Эквивалент в Python |
|-----------------|----------|---------------------|
| `touch file` | Создание пустого файла | `Path.write_text("")` |
| `cp a b` | Копирование | `shutil.copy2(a, b)` |
| `mv a b` | Перемещение / переименование | `shutil.move(a, b)` |
| `rm file` | Удаление | `Path.unlink()` |

Финальная команда `ls -la` подтверждает, что каталог очищен перед переходом к следующей части.

---

### Шаг B2: Минимальный рабочий процесс «редактировать — сохранить — выйти»

При каждом редактировании файла в Vim выполняйте следующие четыре шага:

| Шаг | Клавиша / команда | Что происходит |
|-----|-------------------|----------------|
| 1 | Нажать `i` | Переход в **режим вставки**. В строке состояния отображается `-- INSERT --`. |
| 2 | Ввести (или вставить) содержимое | Текст добавляется в позиции курсора. |
| 3 | Нажать `Esc` | Возврат в **обычный режим**. Строка состояния очищается. |
| 4 | Ввести `:wq` и нажать `Enter` | **Запись** файла на диск и **выход** из Vim. |

**Зачем:** Обычный режим предназначен для навигации и команд; режим вставки — для ввода текста. Перед вводом `:wq` необходимо находиться в обычном режиме.

**Практика:** Введите строку текста, сохраните и выйдите. Затем убедитесь, что файл создан:

```bash
cat notes.txt
```

Удалите тренировочный файл по завершении: `rm notes.txt`

---

### Краткий справочник Vim — сохранение и режимы

| Задача | Команда | Примечание |
|--------|---------|------------|
| Перейти в режим вставки | `i` | Ввод текста с позиции курсора |
| Вставить после курсора | `a` | Альтернатива `i`; вставка после курсора |
| Новая строка ниже | `o` | Открывает новую строку и переходит в режим вставки |
| Вернуться в обычный режим | `Esc` | Всегда нажимайте Esc перед вводом команды |
| Сохранить (записать) | `:w` + `Enter` | Продолжить редактирование после сохранения |
| Сохранить и выйти | `:wq` + `Enter` | Стандартный способ завершения редактирования |
| Выйти без сохранения | `:q!` + `Enter` | Все несохранённые изменения теряются |
| Включить режим вставки | `:set paste` + `Enter` | Предотвращает автоотступ при вставке из буфера обмена |
| Отключить режим вставки | `:set nopaste` + `Enter` | Восстанавливает стандартное поведение отступов в Vim |

> **Совет по вставке:** При вставке многострочного кода из терминала или браузера всегда вводите `:set paste` перед нажатием `i`, вставляйте код, затем нажимайте `Esc` и `:set nopaste`. Это предотвращает некорректное расставление отступов в Vim.

### Краткий справочник Vim — редактирование и поиск

| Вставить после курсора | `p` | Вставляет содержимое регистра |
| Поиск вперёд | `/слово` + `Enter` | Подсвечивает совпадения |
| Перейти к следующему совпадению | `n` | Нажмите ещё раз для продолжения |
| Заменить все вхождения | `:%s/старое/новое/g` + `Enter` | Заменяет каждое вхождение в файле |
| Заменить с подтверждением | `:%s/старое/новое/gc` + `Enter` | Запрашивает подтверждение перед каждой заменой |

## Часть D — Виртуальные среды и воспроизводимая установка (обязательно)

> **Текущее расположение:** `~/lab11`

**Виртуальная среда** (venv) — это самодостаточная установка Python, которая хранит пакеты проекта отдельно от общесистемного Python и других проектов. Это предотвращает конфликты версий.

Файл **`requirements.txt`** фиксирует точные названия и версии пакетов, установленных в виртуальной среде. Любой пользователь может воссоздать ту же среду на другой машине с помощью команды:

```bash
pip install -r requirements.txt
```

Эта команда построчно читает `requirements.txt` и устанавливает каждый перечисленный пакет в активную виртуальную среду.

## Часть G — Соответствие команд Bash и Python (навыки для файлового менеджера)

Следующая таблица сопоставляет распространённые операции оболочки с инструментами стандартной библиотеки Python, используемыми в файловом менеджере. Изучите её перед написанием кода в части I.

| Операция файлового менеджера | Команда Bash | Стандартная библиотека Python |
|------------------------------|--------------|-------------------------------|
| Вывести текущий каталог | `pwd` | `Path.cwd()` |
| Перечислить содержимое каталога | `ls` | `Path.iterdir()` |
| Создать дерево каталогов | `mkdir -p dir` | `Path.mkdir(parents=True, exist_ok=True)` |

| Переместить / переименовать файл | `mv src dst` | `shutil.move(src, dst)` |

## Часть H — Файл конфигурации (config.json) (обязательно)

> **Текущее расположение:** `~/lab11`

### H1) Зачем нужен файл конфигурации?

Файловый менеджер не должен содержать жёстко закодированные пути, кодировку или флаги функций в коде Python. Жёстко закодированные значения требуют изменения кода для корректировки любой настройки. **Файл конфигурации** выносит эти значения наружу: вы изменяете файл, а не код.

В данной лабораторной работе `config.json` задаёт **рабочее пространство** — граничный каталог, внутри которого разрешены все файловые операции. Чтобы указать файловому менеджеру другую папку, достаточно изменить значение `"workspace"` в JSON-файле.

---

### H2) Создание каталога fm_project и файла config.json

**Описание полей:**

| Поле | Тип | Значение |
|------|-----|----------|
| `workspace` | строка | Путь к изолированной папке **относительно `fm_project`**. Все операции I/O ограничены этой папкой. |

| `allow_delete` | булево | Флаг функции. При значении `false` команда `del` отключается. Это позволяет администратору запретить удаление без изменения кода Python. |

**Ссылка:** [json — Кодировщик и декодировщик JSON](https://docs.python.org/3/library/json.html)

#### ПП1 — Шаг 3: Создание файла config_loader.py

> **Расположение:** `~/lab11/fm_project`

**Что нужно сделать:**

1. Откройте Vim: `vim config_loader.py`
2. Нажмите `:set paste` + `Enter`, затем `i`, вставьте код ниже, затем `Esc` + `:set nopaste` + `:wq` + `Enter`.

```python
import json
from dataclasses import dataclass
from pathlib import Path


@dataclass(frozen=True)
class AppConfig:
    workspace: Path       # абсолютный или относительный путь к изолированной папке
    start_dir: str = "."  # начальный подкаталог внутри рабочего пространства
    encoding: str = "utf-8"
    allow_delete: bool = True


def load_config(path: str = "config.json") -> AppConfig:
    """
    Читает config.json из текущего рабочего каталога и возвращает AppConfig.
    Вызывает FileNotFoundError, если файл отсутствует.
    Вызывает ValueError, если обязательное поле 'workspace' отсутствует.
    """
    p = Path(path)
    if not p.exists():
        raise FileNotFoundError(
            f"Config file not found: '{path}'. "
            f"Run this script from the fm_project directory."
        )

    data = json.loads(p.read_text(encoding="utf-8"))

    if "workspace" not in data:
        raise ValueError("config.json must contain the field 'workspace'.")

    return AppConfig(
        workspace=Path(data["workspace"]),
        start_dir=data.get("start_dir", "."),
        encoding=data.get("encoding", "utf-8"),
        allow_delete=bool(data.get("allow_delete", True)),
    )
```

**Зачем:** Отделение загрузки конфигурации от остального кода — это хорошая практика программирования. При изменении пути к рабочему пространству или кодировки вы редактируете `config.json`, а не исходный код Python. Декоратор `@dataclass(frozen=True)` делает `AppConfig` неизменяемым: после загрузки настройки не могут быть случайно изменены во время выполнения.

**Ключевые решения при проектировании:**

- `AppConfig.workspace` является объектом `Path`, а не обычной строкой, что позволяет напрямую вызывать методы `pathlib` (`resolve()`, `mkdir()`).
- `data.get(key, default)` позволяет избежать `KeyError` для необязательных полей, используя безопасные значения по умолчанию.
- Сообщение об ошибке в `FileNotFoundError` точно указывает пользователю, как устранить проблему.
- Значение `workspace` в `config.json` — это путь **относительно текущего рабочего каталога** при запуске программы; всегда запускайте приложение из `fm_project`.

#### ПП1 — Шаг 4: Создание файла safe_paths.py

**Что нужно сделать:**

1. Откройте Vim: `vim safe_paths.py`
2. Нажмите `:set paste` + `Enter`, затем `i`, вставьте код ниже, затем `Esc` + `:set nopaste` + `:wq` + `Enter`.

```python
from pathlib import Path


class Sandbox:
    """
    Ограничивает доступ ко всем путям единственным каталогом рабочего пространства.

    Любой пользовательский путь, разрешающийся за пределами рабочего пространства,
    вызывает PermissionError. Это блокирует атаки обхода каталогов, например
    '../secret.txt' или абсолютные пути вроде '/etc/passwd'.
    """

    def __init__(self, workspace: Path) -> None:
        # resolve() преобразует путь в абсолютный и устраняет '..' и символические ссылки.
        self.workspace: Path = workspace.resolve()
        # Создать рабочее пространство, если оно ещё не существует.
        self.workspace.mkdir(parents=True, exist_ok=True)

    def resolve_inside(self, user_path: str) -> Path:
        """
        Присоединяет user_path к рабочему пространству и разрешает результат.
        Возвращает разрешённый Path, если он находится внутри рабочего пространства.
        Вызывает PermissionError, если он находится снаружи.
        """
        candidate: Path = (self.workspace / user_path).resolve()

        # Путь «находится внутри» рабочего пространства, если рабочее пространство является
        # одним из его предков, либо если кандидат И ЕСТЬ само рабочее пространство.
        if candidate == self.workspace or self.workspace in candidate.parents:
            return candidate

        raise PermissionError(
            f"Access denied: '{user_path}' resolves outside the workspace."
        )
```

**Зачем — объяснение проверки принадлежности:**

`Path.resolve()` преобразует любой путь в абсолютную нормализованную форму:
1. Разворачивает компоненты `..` и `.` (например, `workspace/../secret` → `/home/user/lab11/secret`)
2. Следует по символическим ссылкам к их реальным целям

После разрешения `candidate.parents` представляет упорядоченную последовательность всех каталогов-предков `candidate`. Условие:

```
candidate == self.workspace   ИЛИ   self.workspace in candidate.parents
```

означает: путь-кандидат либо является самим рабочим пространством, либо рабочее пространство присутствует где-то в иерархии предков кандидата — то есть кандидат находится внутри рабочего пространства. Любой путь, не удовлетворяющий этому условию, находится за пределами «песочницы» и отклоняется с исключением `PermissionError`.

```python
import shutil
from pathlib import Path
from safe_paths import Sandbox


class FileOps:
    """
    Предоставляет файловые операции в изолированной среде: чтение, запись, копирование, перемещение, удаление.

    Каждый метод разрешает путь через Sandbox перед выполнением любых операций I/O.
    Операции за пределами рабочего пространства, таким образом, невозможны —
    Sandbox предварительно вызывает PermissionError.
    """

    def __init__(self, sandbox: Sandbox, encoding: str = "utf-8") -> None:
        self.sb = sandbox
        self.encoding = encoding

    def write_text(self, name: str, text: str) -> None:
        """Записывает текст в файл внутри рабочего пространства, создавая родительские каталоги."""
        p = self.sb.resolve_inside(name)
        p.parent.mkdir(parents=True, exist_ok=True)
        p.write_text(text, encoding=self.encoding)

    def read_text(self, name: str) -> str:
        """Читает и возвращает текстовое содержимое файла внутри рабочего пространства."""
        p = self.sb.resolve_inside(name)
        return p.read_text(encoding=self.encoding)

    def copy_file(self, src: str, dst: str) -> None:
        """Копирует src в dst внутри рабочего пространства. Создаёт родительские каталоги dst."""
        s = self.sb.resolve_inside(src)
        d = self.sb.resolve_inside(dst)
        d.parent.mkdir(parents=True, exist_ok=True)
        shutil.copy2(s, d)

    def move(self, src: str, dst: str) -> None:
        """Перемещает (переименовывает) src в dst внутри рабочего пространства."""
        s = self.sb.resolve_inside(src)
        d = self.sb.resolve_inside(dst)
        d.parent.mkdir(parents=True, exist_ok=True)
        shutil.move(str(s), str(d))

    def delete_file(self, name: str) -> None:
        """Удаляет единственный файл внутри рабочего пространства."""
        p = self.sb.resolve_inside(name)
        p.unlink()
```

**Зачем:**

- Каждый метод вызывает `self.sb.resolve_inside(...)` в качестве первого действия. Безопасность путей обеспечивается в одном центральном месте (Sandbox), а не дублируется в каждом методе.
- `p.parent.mkdir(parents=True, exist_ok=True)` автоматически создаёт все отсутствующие промежуточные каталоги.
- `shutil.copy2` сохраняет метаданные исходного файла (временны́е метки, права доступа). `shutil.move` используется вместо `Path.rename`, поскольку корректно работает между разными точками монтирования файловой системы.
- `str(s)` и `str(d)` передаются в `shutil.move` для совместимости с Python 3.7 и более ранними версиями.

##### ПП3A — Шаг 1: Создание файла cli_manager.py

> **Расположение:** `~/lab11/fm_project`

**Что нужно сделать:**

1. Откройте Vim: `vim cli_manager.py`
2. Вставьте содержимое ниже, сохраните и выйдите.

```python
from config_loader import load_config
from safe_paths import Sandbox
from file_ops import FileOps


HELP = """
Commands:
  put  <file> <text>   Write text to a file (creates the file if absent)
  read <file>          Print the contents of a file
  copy <src> <dst>     Copy src to dst
  move <src> <dst>     Move (rename) src to dst
  del  <file>          Delete a file  (only if allow_delete = true in config)
  help                 Show this message
  exit                 Quit the file manager
"""


def main() -> None:
    cfg = load_config("config.json")
    sb  = Sandbox(cfg.workspace)
    ops = FileOps(sb, encoding=cfg.encoding)

    print("Simple File Manager")
    print(f"Workspace : {sb.workspace}")
    print(f"Encoding  : {cfg.encoding}")
    print(f"Delete    : {'enabled' if cfg.allow_delete else 'DISABLED'}")
    print(HELP)

    while True:
        try:
            cmdline = input("fm> ").strip()
        except (EOFError, KeyboardInterrupt):
            print("\nGoodbye")
            return

        if not cmdline:
            continue

        parts = cmdline.split(" ", 2)
        cmd   = parts[0].lower()

        try:
            if cmd == "exit":
                print("Goodbye")
                return

            elif cmd == "help":
                print(HELP)

            elif cmd == "put":
                if len(parts) < 2:
                    print("Usage: put <file> <text>")
                    continue
                filename = parts[1]
                text     = parts[2] if len(parts) > 2 else ""
                ops.write_text(filename, text)
                print(f"Saved: {filename}")

            elif cmd == "read":
                if len(parts) < 2:
                    print("Usage: read <file>")
                    continue
                print(ops.read_text(parts[1]))

            elif cmd == "copy":
                if len(parts) < 3:
                    print("Usage: copy <src> <dst>")
                    continue
                _, src, dst = cmdline.split(" ", 2)
                ops.copy_file(src, dst)
                print(f"Copied: {src} -> {dst}")

            elif cmd == "move":
                if len(parts) < 3:
                    print("Usage: move <src> <dst>")
                    continue
                _, src, dst = cmdline.split(" ", 2)
                ops.move(src, dst)
                print(f"Moved: {src} -> {dst}")

            elif cmd == "del":
                if not cfg.allow_delete:
                    print("Delete is disabled (allow_delete = false in config.json).")
                    continue
                if len(parts) < 2:
                    print("Usage: del <file>")
                    continue
                ops.delete_file(parts[1])
                print(f"Deleted: {parts[1]}")

            else:
                print(f"Unknown command: '{cmd}'. Type 'help' for the command list.")

        except PermissionError as e:
            print(f"Security error: {e}")
        except FileNotFoundError as e:
            print(f"File not found: {e}")
        except Exception as e:
            print(f"Error: {e}")


if __name__ == "__main__":
    main()
```

**Зачем — ключевые архитектурные решения:**

| Особенность | Причина |
|-------------|---------|
| `HELP` как константа уровня модуля | Легко обновлять; выводится при запуске и по команде `help` |
| Проверка `len(parts) < N` перед каждой командой | Предотвращает `IndexError`, когда пользователь опускает обязательные аргументы |
| `except (EOFError, KeyboardInterrupt)` вокруг `input()` | Позволяет корректно выйти по `Ctrl+D` или `Ctrl+C` |
| Три отдельных блока `except` | Различает ошибки безопасности, отсутствующие файлы и другие ошибки времени выполнения |
| Защита `if __name__ == "__main__"` | Позволяет импортировать модуль в тестах без запуска цикла |
